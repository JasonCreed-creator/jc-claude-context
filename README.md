# M&C 회의록 관리 대시보드

> **TMS "회의록 관리" 모듈 통합 예정** — 본 대시보드는 사내 TMS(tms.mnccom.com)의 "회의록 관리" 메뉴에 통합될 시안입니다. 현재 단계는 **전사 공유 후 피드백 수집**.

회사 전체 주간/월례 회의록을 부서별로 작성·관리하고, KPI·진행률을 자동 집계하는 협업형 회의록 대시보드.

## 전사 공유 안내 (현재 단계)

- `dashboard.html`은 데모용 샘플 데이터로 동작합니다 (사외비 정보는 모두 마스킹된 2026-05-11 주간회의 5개 부서)
- 회의록 작성 / 부서별 분배 / 액션 토글 / 참석자 체크 / 부서별 프로젝트 필터 UX를 실제로 클릭하며 검토 가능
- 피드백: 디자인·정보 구조·운영 흐름 의견 환영

## 다음 단계 로드맵
- **Phase A** (현재): 운영 UX(입력·CRUD·체크박스 토글·부서 분배)
- **Phase B**: JC Design System 채택 + KPI Strip 5개 카드 + D-day 카드
- **Phase C**: 프로젝트 라이프사이클 칸반 + 입찰 파이프라인 + RMB Funnel
- **Phase D**: 월례회의 섹션 + TMS 모듈 정식 통합

- **스택**: 정적 HTML/CSS/JS (의존성 0, 빌드 도구 없음)
- **데이터**: 구글시트(gviz/tq JSON API) → 30초 폴링
- **입력**: 구글폼 또는 구글시트 동시편집
- **배포**: GitHub Pages

## 폴더 구조

```
docs/
├── index.html              대시보드 본체
├── config.json             시트 ID·컬럼 매핑·링크
├── sample-data.json        시트 미연결 시 fallback
└── assets/
    ├── css/style.css       디자인 토큰·반응형·다크모드
    └── js/
        ├── markdown.js     의존성 0 미니 마크다운 파서
        ├── sheet.js        gviz/tq fetch + 30초 폴링
        ├── filter.js       주차·프로젝트·키워드 필터
        ├── render.js       KPI·리스트·상세 렌더
        └── app.js          메인 컨트롤러
```

## 빠른 시작 (로컬 검증)

```bash
cd docs
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

`config.json`의 `sheet.id`가 비어있으면 `sample-data.json`(더미 회의록 3건)으로 동작합니다.

## 더 간단한 방법 — `dashboard.html` 단일 파일

루트의 **`dashboard.html`** 파일은 모든 자원(CSS·JS·샘플데이터)을 인라인으로 포함한 standalone 버전입니다. 별도 서버·빌드·연결 없이:

1. `dashboard.html`을 다운로드 (GitHub에서 `Raw` → 우클릭 `다른 이름으로 저장`)
2. 파일을 **더블클릭** → 브라우저에서 즉시 열림
3. 샘플 회의록 3건이 표시되며 모든 기능 동작

### 회의록 입력 방법 (2가지)

**A. 로컬 입력 모드** — 셋업 0분, 혼자 즉시 사용
- 우상단 `＋ 회의록 작성` 클릭 → 모달에서 입력 → **저장**
- 브라우저 localStorage에 저장 (해당 PC·브라우저에서만 유지)
- 상세 패널에서 ✎ 편집 / 🗑 삭제 가능
- `↓ Export` / `↑ Import`로 JSON 백업·복원

**B. 시트 연동 모드** — 5인 협업, 10분 셋업
- 상세 가이드: [SETUP-SHEET.md](SETUP-SHEET.md)
- 구글시트 + 구글폼 셋업 후 `STANDALONE_CONFIG`에 시트 ID 입력
- 30초 폴링으로 모든 PC에 자동 반영

> 둘 다 동시에 사용 가능. 시트 데이터 + 로컬 데이터가 한 화면에 함께 표시되며, 각 회의록 카드에 `로컬 저장` 태그로 구분됩니다.

## 구글시트 연결 (운영 전환)

### 1. 구글시트 양식 만들기

새 구글시트를 만들고 1행에 다음 헤더를 입력하세요. **순서는 자유**, 라벨은 정확히 일치해야 합니다(`config.json`의 `column_mapping` 값).

| 헤더 | 설명 | 예시 |
|---|---|---|
| 타임스탬프 | 자동 (구글폼 사용 시) | 2026-05-11 10:32 |
| 회의일자 | 날짜 | 2026-05-11 |
| 주차 | 수식 자동 | 2026-W20 |
| 회의명 | 텍스트 | 신사업실 주간 정기회의 |
| 프로젝트 | 객관식 | MiceConfigurator / 리멤버 / ConfEx / IUCR / 기타 |
| 작성자 | 객관식 | 이진철·김정길·곽은지·김현·이지후 |
| 참석자 | 텍스트 | 이진철, 김정길, … |
| 안건 | 마크다운 장문 | `## 안건\n- …` |
| 결정사항 | 마크다운 장문 | `## 결정\n- …` |
| 액션아이템 | 마크다운 장문 | `- [ ] 곽은지: …`, `- [x] 완료된 일` |
| 다음회의 | 날짜 | 2026-05-18 |
| 비고 | 텍스트 | 보조 메모 |

**주차 자동 수식**(C2 셀):
```
=IF(B2="","",YEAR(B2)&"-W"&TEXT(WEEKNUM(B2,2),"00"))
```

**액션아이템 체크박스 규칙**:
- `- [ ] 할 일` → 미완료
- `- [x] 한 일` → 완료
- 대시보드의 "완료율" KPI는 이 체크박스 비율로 자동 계산됩니다.

### 2. 시트 공개 설정

상단 **공유** → **링크가 있는 모든 사람** → **편집자**로 설정.

> 사내 정보 노출 우려가 있는 경우, 민감 정보는 시트에 쓰지 마시고 별도 비공개 문서에 보관하세요.

### 3. config.json 업데이트

`docs/config.json`을 수정:

```json
{
  "sheet": {
    "id": "여기에_시트_ID_붙여넣기",
    "gid": "0",
    "sheet_name": "회의록"
  },
  "links": {
    "form_url": "https://forms.gle/your-form",
    "edit_url": "https://docs.google.com/spreadsheets/d/.../edit"
  }
}
```

시트 ID는 URL `https://docs.google.com/spreadsheets/d/{여기}/edit`에서 추출.

### 4. 구글폼 만들기 (선택)

폼을 만들 때 각 질문 제목을 시트 헤더와 동일하게 작성하고, **응답 → 스프레드시트로 연결**을 기존 시트로 지정하세요. 폼 응답이 자동으로 시트에 적재됩니다.

폼 공개 링크는 `config.json`의 `links.form_url`에 입력하면 대시보드 상단의 **＋ 회의록 작성** 버튼이 활성화됩니다.

## GitHub Pages 배포

저장소 설정 → **Pages** → Branch: `main`(또는 운영 브랜치) / Folder: `/docs` → Save.

수 분 후 `https://JasonCreed-creator.github.io/jc-claude-context/` 에서 접속 가능.

> **주의**: 저장소가 Private인 경우 GitHub Pages는 유료 플랜 또는 Public 전환이 필요합니다.

## 회의 운영 흐름

1. 회의 시작 → 회의실 큰 화면에 대시보드 띄움 (자동 30초 갱신)
2. 작성자가 노트북에서 **＋ 회의록 작성** 버튼 클릭 → 구글폼 작성
3. 동시에 5인이 시트 동시편집으로 액션아이템 추가/체크
4. 30초 안에 대시보드 KPI·리스트·상세에 자동 반영

## FAQ — 자주 묻는 운영 질문

**Q. 회의 중에 작성자가 추가하면 다른 사람 화면에는 언제 반영되나요?**  
A. 최대 30초. `config.json`의 `app.polling_interval_ms`로 조정 가능. 즉시 보고 싶으면 우측 상단 `↻ 새로고침` 버튼.

**Q. 시트에 비밀번호나 계약금액 같은 민감정보가 들어가면?**  
A. 시트가 "링크 있는 모든 사람 편집"이므로 누구나 열 수 있습니다. 민감정보는 별도 비공개 문서로 분리하세요. 회의록 본문에 마스킹(`****`)을 권장.

**Q. 액션아이템 체크박스 형식을 잘못 쓰면?**  
A. `- [ ]` (미완료), `- [x]` (완료) 이외 형식은 일반 리스트로 처리됩니다(완료율 집계 제외).

**Q. 시트 컬럼 순서를 바꿔도 되나요?**  
A. 네. 헤더 라벨(`config.json`의 `column_mapping` 값) 기준으로 매핑하므로 순서 무관.

**Q. 시트 헤더 라벨을 바꾸려면?**  
A. `docs/config.json`의 `column_mapping`에서 표준 키의 값을 새 라벨로 변경 + 커밋.

**Q. 작성자·프로젝트 목록을 늘리고 싶을 때?**  
A. `docs/config.json`의 `team.members`(작성자), `projects`(프로젝트 태그) 배열을 수정. 칩 자동 갱신.

**Q. 시트 연결 전 미리 보고 싶어요.**  
A. `config.json`의 `sheet.id`를 비워두면 `sample-data.json`(더미 3건)으로 동작합니다.

**Q. 외부에 노출하기 싫은데 회의실 5인 전용으로만 운영 가능?**  
A. 두 가지 방법:
  1. GitHub Pages 대신 회의실 PC 1대에 `python3 -m http.server` 띄우고 사내망에서만 접속
  2. 저장소를 Public으로 두되, 시트 자체를 비공개 + Apps Script로 토큰 검증 API 구축(3턴 백로그)
