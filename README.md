# M&C 신사업실 주간회의록 대시보드

회의실에서 큰 화면으로 띄우고, 5인이 각자 노트북에서 동시에 기록하는 협업형 회의록 대시보드.

- **스택**: 정적 HTML/CSS/JS (의존성 0, 빌드 도구 없음)
- **데이터**: 구글시트(gviz/tq JSON API) → 30초 폴링
- **입력**: 구글폼 또는 구글시트 동시편집
- **배포**: GitHub Pages

## 폴더 구조

```
docs/
├── index.html              대시보드 본체
├── config.json             시트 ID·컬럼 매핑·링크
├── sample-data.json        시트 미연결 시 fallback
└── assets/
    ├── css/style.css       디자인 토큰·반응형·다크모드
    └── js/
        ├── markdown.js     의존성 0 미니 마크다운 파서
        ├── sheet.js        gviz/tq fetch + 30초 폴링
        ├── filter.js       주차·프로젝트·키워드 필터
        ├── render.js       KPI·리스트·상세 렌더
        └── app.js          메인 컨트롤러
```

## 빠른 시작 (로컬 검증)

```bash
cd docs
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

`config.json`의 `sheet.id`가 비어있으면 `sample-data.json`(더미 회의록 3건)으로 동작합니다.

## 구글시트 연결 (운영 전환)

### 1. 구글시트 양식 만들기

새 구글시트를 만들고 1행에 다음 헤더를 입력하세요. **순서는 자유**, 라벨은 정확히 일치해야 합니다(`config.json`의 `column_mapping` 값).

| 헤더 | 설명 | 예시 |
|---|---|---|
| 타임스탬프 | 자동 (구글폼 사용 시) | 2026-05-11 10:32 |
| 회의일자 | 날짜 | 2026-05-11 |
| 주차 | 수식 자동 | 2026-W20 |
| 회의명 | 텍스트 | 신사업실 주간 정기회의 |
| 프로젝트 | 객관식 | MiceConfigurator / 리멤버 / ConfEx / IUCR / 기타 |
| 작성자 | 객관식 | 이진철·김정길·곽은지·김현·이지후 |
| 참석자 | 텍스트 | 이진철, 김정길, … |
| 안건 | 마크다운 장문 | `## 안건\n- …` |
| 결정사항 | 마크다운 장문 | `## 결정\n- …` |
| 액션아이템 | 마크다운 장문 | `- [ ] 곽은지: …`, `- [x] 완료된 일` |
| 다음회의 | 날짜 | 2026-05-18 |
| 비고 | 텍스트 | 보조 메모 |

**주차 자동 수식**(C2 셀):
```
=IF(B2="","",YEAR(B2)&"-W"&TEXT(WEEKNUM(B2,2),"00"))
```

**액션아이템 체크박스 규칙**:
- `- [ ] 할 일` → 미완료
- `- [x] 한 일` → 완료
- 대시보드의 "완료율" KPI는 이 체크박스 비율로 자동 계산됩니다.

### 2. 시트 공개 설정

상단 **공유** → **링크가 있는 모든 사람** → **편집자**로 설정.

> 사내 정보 노출 우려가 있는 경우, 민감 정보는 시트에 쓰지 마시고 별도 비공개 문서에 보관하세요.

### 3. config.json 업데이트

`docs/config.json`을 수정:

```json
{
  "sheet": {
    "id": "여기에_시트_ID_붙여넣기",
    "gid": "0",
    "sheet_name": "회의록"
  },
  "links": {
    "form_url": "https://forms.gle/your-form",
    "edit_url": "https://docs.google.com/spreadsheets/d/.../edit"
  }
}
```

시트 ID는 URL `https://docs.google.com/spreadsheets/d/{여기}/edit`에서 추출.

### 4. 구글폼 만들기 (선택)

폼을 만들 때 각 질문 제목을 시트 헤더와 동일하게 작성하고, **응답 → 스프레드시트로 연결**을 기존 시트로 지정하세요. 폼 응답이 자동으로 시트에 적재됩니다.

폼 공개 링크는 `config.json`의 `links.form_url`에 입력하면 대시보드 상단의 **＋ 회의록 작성** 버튼이 활성화됩니다.

## GitHub Pages 배포

저장소 설정 → **Pages** → Branch: `main`(또는 운영 브랜치) / Folder: `/docs` → Save.

수 분 후 `https://JasonCreed-creator.github.io/jc-claude-context/` 에서 접속 가능.

> **주의**: 저장소가 Private인 경우 GitHub Pages는 유료 플랜 또는 Public 전환이 필요합니다.

## 회의 운영 흐름

1. 회의 시작 → 회의실 큰 화면에 대시보드 띄움 (자동 30초 갱신)
2. 작성자가 노트북에서 **＋ 회의록 작성** 버튼 클릭 → 구글폼 작성
3. 동시에 5인이 시트 동시편집으로 액션아이템 추가/체크
4. 30초 안에 대시보드 KPI·리스트·상세에 자동 반영
