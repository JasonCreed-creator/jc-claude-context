# 시트 연동 셋업 가이드 (10분 완료)

`dashboard.html`을 시트와 연결하면 5인이 동시에 입력해도 30초 안에 모든 화면에 반영됩니다. 로컬 입력 모드와 병행 가능합니다.

---

## STEP 1 · 구글시트 만들기 (3분)

1. https://sheets.google.com 접속 → **빈 스프레드시트** 클릭
2. 시트 이름을 **`회의록`**으로 변경 (좌측 하단 탭 이름)
3. 1행에 아래 헤더를 그대로 입력 (순서·라벨 정확히 일치):

```
타임스탬프 | 회의일자 | 주차 | 회의명 | 프로젝트 | 작성자 | 참석자 | 안건 | 결정사항 | 액션아이템 | 다음회의 | 비고
```

복사용 (탭 구분):
```
타임스탬프	회의일자	주차	회의명	프로젝트	작성자	참석자	안건	결정사항	액션아이템	다음회의	비고
```

4. **주차 자동 수식**: C2 셀에 입력 → 채우기 핸들로 아래까지 드래그
```
=IF(B2="","",YEAR(B2)&"-W"&TEXT(WEEKNUM(B2,2),"00"))
```

5. 상단 메뉴 **공유** → **링크가 있는 모든 사람** → 권한 **편집자** 선택 → 완료

---

## STEP 2 · 시트 ID 추출 + dashboard.html 수정 (2분)

시트 URL은 다음 형태입니다:
```
https://docs.google.com/spreadsheets/d/1AbC...xYz/edit#gid=0
                                       └────┬────┘
                                          시트 ID
```

`dashboard.html`을 메모장·VS Code 등으로 열고 **`STANDALONE_CONFIG`**를 찾아 수정:

```js
window.STANDALONE_CONFIG = {
  app: { ... },
  sheet: {
    id: '1AbC...xYz',     // ← 시트 ID 붙여넣기
    gid: '0',
    sheet_name: '회의록'
  },
  links: {
    form_url: '',          // 폼 만들면 여기에 (STEP 3)
    edit_url: ''           // 비워두면 시트 ID로 자동 생성
  },
  ...
};
```

저장 후 더블클릭 → 30초 안에 시트 데이터로 전환됩니다. 우측 상단 **시트 편집** 버튼이 자동으로 활성화됩니다.

---

## STEP 3 · 구글폼 만들기 (선택, 5분)

회의실에서 5인이 각자 노트북으로 회의록을 입력하려면 폼이 가장 편합니다.

1. https://forms.google.com → **빈 양식**
2. 질문을 다음 순서·유형으로 추가:

| 질문 제목 | 유형 | 필수 |
|---|---|:-:|
| 회의일자 | 날짜 | ✓ |
| 회의명 | 단답형 | ✓ |
| 프로젝트 | 드롭다운 (MiceConfigurator/리멤버/ConfEx/IUCR/기타) | ✓ |
| 작성자 | 드롭다운 (이진철/김정길/곽은지/김현/이지후) | ✓ |
| 참석자 | 단답형 | |
| 안건 | 장문형 | |
| 결정사항 | 장문형 | |
| 액션아이템 | 장문형 | |
| 다음회의 | 날짜 | |
| 비고 | 장문형 | |

> 폼 질문 제목은 시트 헤더 라벨과 **반드시 동일**해야 합니다(자동 매핑).

3. 상단 **응답 탭** → 초록 시트 아이콘 → **기존 스프레드시트 선택** → STEP 1에서 만든 시트 지정
4. 상단 **보내기** → 링크 아이콘 → URL 복사
5. `dashboard.html`의 `STANDALONE_CONFIG.links.form_url`에 붙여넣기 저장

이후 대시보드의 **＋ 회의록 작성** 버튼이 폼을 새 탭으로 열어줍니다.

---

## STEP 4 · 회의 운영 흐름

1. 회의 시작 → 회의실 큰 화면에 `dashboard.html` 열기 (자동 30초 폴링)
2. 5인이 각자 노트북에서:
   - **공식 회의록**: ＋ 회의록 작성 → 구글폼 입력 → 시트 적재
   - **즉석 메모**: 대시보드에서 직접 ＋ 회의록 작성 → 로컬 저장 (localStorage)
3. 30초 안에 시트 데이터가 모든 화면에 반영
4. 회의 후 시트에서 액션아이템 체크박스 `- [ ]` → `- [x]`로 업데이트 → 완료율 KPI 자동 갱신

---

## 로컬 모드 vs 시트 모드 비교

| 항목 | 로컬 입력 모드 | 시트 연동 모드 |
|---|---|---|
| 셋업 | 0분 (즉시 사용) | 10분 |
| 5인 동시 입력 | ❌ | ✅ |
| PC 간 동기화 | ❌ (해당 브라우저만) | ✅ |
| 자동 백업 | ❌ (JSON Export로 수동) | ✅ (구글 자동) |
| 편집/삭제 | ✅ (대시보드에서) | 시트에서 직접 |
| 둘 다 사용 | 한 화면에 함께 표시 (`로컬 저장` 태그) | |

## 자주 묻는 질문

**Q. 시트 연결 후에도 로컬 입력은 유지되나요?**  
A. 네. 시트 데이터와 로컬 데이터가 한 화면에 함께 표시됩니다. 로컬은 우상단 `↓ Export` 버튼으로 JSON 백업 가능.

**Q. 로컬 데이터를 시트로 옮기려면?**  
A. `↓ Export`로 JSON 저장 후, 시트에 수동 복사 또는 Apps Script로 일괄 import. (자동 마이그레이션은 백로그)

**Q. 시트 권한을 좁히고 싶어요. "보기만" 가능한 사람도 받을 수 있나요?**  
A. 시트는 **편집자** 권한이 필요합니다(폼 적재 + 동시편집). 시트 자체에 민감정보를 쓰지 않는 게 안전.

**Q. 시트 컬럼을 추가하고 싶을 때?**  
A. `STANDALONE_CONFIG.column_mapping`에 새 키 추가 + 렌더링 코드 일부 수정 필요. 요청 주세요.

**Q. 액션아이템 완료율이 이상해요.**  
A. 체크박스는 반드시 `- [ ]` (미완료) / `- [x]` (완료) 형식. 대소문자 X도 인식. 그 외는 일반 리스트로 처리됨.
